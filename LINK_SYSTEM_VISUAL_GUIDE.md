## ğŸ”„ Ordered Pair Logic

```
Problem: Prevent duplicate mutuals/matches

Example:
  User A (uuid: aaa...) and User B (uuid: bbb...)
  
  Without ordering:
    Row 1: (profile_a: aaa, profile_b: bbb) âœ“
    Row 2: (profile_a: bbb, profile_b: aaa) âœ“ DUPLICATE!
  
  With ordering (LEAST/GREATEST):
    Row 1: (profile_a: aaa, profile_b: bbb) âœ“
    Row 2: (profile_a: aaa, profile_b: bbb) âœ— UNIQUE CONSTRAINT
  
  Implementation:
    INSERT INTO link_mutuals (profile_a, profile_b)
    VALUES (
      LEAST(user1_id, user2_id),      -- Always smaller UUID
      GREATEST(user1_id, user2_id)    -- Always larger UUID
    );
  
  Lookup (works both ways):
    SELECT * FROM link_mutuals
    WHERE (profile_a = user1 AND profile_b = user2)
       OR (profile_a = user2 AND profile_b = user1);
    
    -- But our index uses LEAST/GREATEST so this is efficient:
    WHERE profile_a = LEAST(user1, user2)
      AND profile_b = GREATEST(user1, user2);
```
